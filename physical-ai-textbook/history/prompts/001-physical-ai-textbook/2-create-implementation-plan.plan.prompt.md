- Initialize a Git repository: `physical-ai-humanoid-robotics`

- Package manager: npm or pnpm (consistent across repo)

  - Language: TypeScript
  - Docs-only mode (no blog)

### 1.2 Core Configuration (`docusaurus.config.ts`)
- Site title: Physical AI & Humanoid Robotics
- Tagline: AI Systems in the Physical World
- Base URL and deployment settings:
  - `url`: GitHub Pages domain
  - `baseUrl`: `/physical-ai-humanoid-robotics/`
- Enable:
  - Docs sidebar
  - MDX support
  - Syntax highlighting for Python, YAML, Bash
- Disable unused features (blog, pages) unless explicitly required
- Add math and diagram support (KaTeX, Mermaid) if needed

### 1.3 Documentation Configuration
- Single docs instance (no multi-instance complexity)
- Sidebar autogenerated but grouped manually by module
- Edit URL linked to GitHub repository
- Versioning disabled for first release

### 1.4 GitHub Pages Deployment
- GitHub Actions workflow:
  - Install dependencies
  - Build Docusaurus site
  - Deploy to `gh-pages` branch
- Ensure reproducible builds (lockfile committed)

## 2. Content Development Phases

### Phase 1: Specification Authoring (Spec-First)
- Create detailed chapter specs using Spec-Kit Plus
- One spec file per chapter
- Specs must define:
  - Learning objectives
  - Section breakdown
  - Labs and simulations
  - AI-agent prompts
- No prose written before spec approval

### Phase 2: Prose Generation (Claude Code)
- Generate MDX content strictly from approved specs
- Maintain consistent tone, terminology, and structure
- Enforce citation rules and academic rigor
- Validate content against constitution constraints

### Phase 3: Technical Integration
- Insert code blocks, callouts, and diagrams
- Validate MDX rendering locally
- Ensure internal links and sidebar navigation work correctly

### Phase 4: Review & Quality Assurance
- Plagiarism and citation checks
- Consistency review across chapters
- Build and deploy preview site
- Fix formatting, hierarchy, and navigation issues

### Phase 5: Final Deployment
- Tag release version
- Deploy to GitHub Pages
- Freeze content for academic use
